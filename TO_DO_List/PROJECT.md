# TO_DO_List

Android-приложение для управления задачами с локальным хранением данных

## О проекте

Проект реализует базовый, но полноценный workflow работы с задачами:
- создание и редактирование задач;
- управление статусом выполнения;
- удаление с быстрым действием;
- фильтрация по состоянию;
- сохранение состояния между перезапусками приложения.

## Основные возможности

- Добавление задач с заголовком, описанием и приоритетом
- Редактирование существующих задач
- Отметка задач как выполненных и возврат в работу
- Удаление задач из списка
- Фильтрация: все / активные / выполненные
- Автосохранение данных в локальной базе Room

## Технологии и стек

- Язык: `Kotlin`
- UI: `Material 3`, `RecyclerView`, `ViewBinding`
- Архитектура: `ViewModel + Repository`
- Данные: `Room (SQLite)`
- Асинхронность: `Kotlin Coroutines`
- Сборка: `Gradle Kotlin DSL`

## Архитектура

В проекте используется разделение ответственности:

- `ui` — экраны, адаптеры, взаимодействие с пользователем
- `viewmodel` — логика представления и управление состоянием экрана
- `repository` — единая точка доступа к данным
- `data/model` — модели домена
- `data/repository` — Room DAO и база данных

Это упрощает сопровождение, тестирование и масштабирование проекта.

## Пользовательские сценарии

1. Пользователь открывает список задач.
2. Создаёт новую задачу через экран добавления.
3. Назначает приоритет и сохраняет.
4. Отмечает задачу выполненной или возвращает в работу.
5. При необходимости удаляет задачу.
6. После перезапуска приложения данные остаются сохранёнными.
